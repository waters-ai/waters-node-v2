## Что сделано

### 1. Redis (store.rs) — фундамент памяти
- multi-DB (0-15): системная, группы 1-6, LLM кэш
- PUBLISH / SUBSCRIBE для стриминга токенов
- XADD / XREAD для Redis Streams (журналы, чаты, findings)
- HSET / HGET для состояния агентов
- StreamSubscriber — подписка на каналы для SSE

### 2. Token streaming (tui_agent.rs)
- Вместо mpsc → PUBLISH в `channel:stream:{session}`
- Параллельно XADD в `stream:tokens:{session}` для истории
- DB выбирается по `group_id` (1-6)

### 3. Веб-дашборд (api.rs)
- SSE endpoint `/api/v1/stream/{session}`
- Статус Redis в дашборде
- Голосовое управление: 🎤 рация (0 токенов) + 🤖 агент-голос
- 6 голосовых профилей для разных агентов

### 4. Skill система (skill.rs)
- SkillManifest расширен: role, category, llm, tools, imported_from
- Рекурсивная загрузка из папок
- `merge_agents()` — слияние двух агентов в третьего
- `summary_for_llm()` — сводка для LLM

### 5. Агенты (agents/)
- 7 TUI-code (совместимость 1:1)
- 10 WATERS профессий (collector, scout, analyst, synthesizer, coordinator, archivist, camera-operator, video-editor, lab-operator, specialist)
- 2 импортированных (TUI + Claude)
- 1 merged (collector + analyst)
- **Всего: 20+ агентов**

### 6. SubAgent runtime (subagent.rs)
- 95 → 500+ строк
- `agent_open()` — создание агента в Redis + tokio task
- `agent_eval()` — чтение состояния из Redis
- `agent_close()` — завершение + journal
- `agent_send_input()` — отправка mid-flight сообщений
- `agent_assign()` — смена задачи на ходу
- Concurrency cap (max 10)
- Background runtime
- Cancellation cascade (закрытие детей)

### 7. Импорт/экспорт агентов (bridge_agent.rs)
- TUI SKILL.md → наш формат
- Claude CLAUDE.md → наш формат
- Cursor .cursorrules → наш формат
- Экспорт в TUI/Claude/Cursor/WATERS
- Массовый импорт из директории

### 8. MCP-сервер (mcp_server.rs)
- tools/list → все агенты как инструменты
- tools/call → вызов агента по имени
- resources/read → состояние ноды
- Порт = HTTP порт + 100

### 9. Рейтинг + безопасность (agent_rating.rs)
- Рейтинг агентов (score, votes, rank)
- YASA-досмотр: проверка bridges, prompt, imported_from
- Топ агентов

### 10. Медиа-мосты (media_bridge.rs)
- NDI output (кадры → сеть)
- OBS WebSocket (переключение сцен)
- RTMP стриминг (YouTube/Twitch)
- HDMI вывод (SDL2/framebuffer)
- Studio feedback (захват → LLM)

### 11. Групповой чат (group_chat.rs)
- Общий чат группы (6 нод + хозяин)
- Подзадачные чаты
- Мнения агентов о задачах
- Redis Stream: `group:{id}:chat`

### 12. Голосовое управление
- 🎤 Рация: MediaRecorder → SSE → AudioContext (0 токенов)
- 🤖 Агент-голос: SpeechRecognition → LLM → SpeechSynthesis
- 6 голосовых профилей (👩👨🧑)
- Переключение пола/языка

### 13. Инфраструктура
- **177 сервер:** waters-node развёрнут, 20+ агентов, Redis
- **3 ноды:** 177 + дача (Win) + дом (Ubuntu)
- **P2P:** /connect для соединения
- **GitHub репозиторий:** https://github.com/waters-ai/waters-node
- **Release v0.4:** бинарник 11MB + документация
