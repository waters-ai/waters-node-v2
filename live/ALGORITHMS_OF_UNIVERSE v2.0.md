## Часть 1. Математический фундамент: от фрактальных метафор к строгим мерам сложности

### 1.1. Отказ от D_f как доказательной метрики

В версии 1.0.1 центральной мерой сложности знания была фрактальная размерность D_f, вычисляемая методом box-counting по конечному облаку точек. Математический анализ выявил фундаментальные ограничения этого подхода:

1. **Для конечного множества точек D_f строго равна нулю** при ε → 0. Любая ненулевая оценка является артефактом выбора диапазона масштабов, а не свойством самого знания.
2. **Box-counting в высоких размерностях (d > 100) статистически несостоятелен** без экспоненциального числа точек из-за «проклятия размерности».
3. **Два принципиально разных состояния могут иметь одинаковую D_f**, что делает псевдометрику d_f(S₁, S₂) = |D_f(S₁) − D_f(S₂)| непригодной для доказательства сходимости.

**Решение:** Мы заменяем D_f на две строго определённые и вычислимые меры:
- **Intrinsic Dimension (ID)** латентного многообразия — для измерения структурной сложности знаний.
- **Свободную энергию F (Free Energy)** — как глобальный функционал, монотонно убывающий при обучении и гарантирующий сходимость к истине.

D_f сохраняется как эвристический индикатор в пользовательском интерфейсе (TUI Хозяина), но исключается из всех математических доказательств.

### 1.2. Intrinsic Dimension (ID) как мера сложности знаний

#### 1.2.1. Теоретическое основание: ε-энтропия по Колмогорову–Тихомирову

Фундаментальной работой советской математической школы является теория ε-энтропии функциональных классов (Колмогоров, Тихомиров, 1959). Для вполне ограниченного множества A в метрическом пространстве определяется:

H_ε(A) = log₂ N_ε(A)


где N_ε(A) — минимальное число точек в ε-сети. **Метрическая размерность** (ёмкостная размерность, или upper box dimension) определяется как:

dim_M(A) = lim sup_{ε→0} H_ε(A) / log₂(1/ε)


Для самоподобных (фрактальных) множеств этот предел существует и совпадает с размерностью Минковского. Однако для реальных данных мы имеем дело с **конечными выборками** с аттрактора скрытой динамической системы. В этом случае мы используем понятие **Intrinsic Dimension** — размерности многообразия, на котором или вблизи которого лежат данные.

#### 1.2.2. Вычислительный метод: локальный PCA с адаптивным радиусом (Фукунага–Олсен–Погорелов)

Метод TwoNN (Facco et al., 2017) является популярным, но неустойчивым при малых выборках и высоких размерностях. Мы применяем **робастный метод локального PCA**, восходящий к работам Фукунаги и Олсена (1971) и обобщённый В. А. Погореловым (1985) для некорректно поставленных задач.

**Алгоритм оценки ID:**
1. Для каждой точки z_i в латентном пространстве R^d (d = 1280 для главной модели LeWM, 192 для легковесных) строим k ближайших соседей (k ≈ 50–100).
2. Вычисляем локальную ковариационную матрицу C_i размера d×d по этим соседям.
3. Находим сингулярные числа σ₁ ≥ σ₂ ≥ ... ≥ σ_d матрицы C_i.
4. Определяем число значимых сингулярных чисел, превышающих порог τ. Порог определяется из распределения Марченко–Пастура для шумовых компонент.
5. Локальная размерность ID_i = #{j : σ_j > τ}. Глобальная ID — медиана локальных оценок.

**Свойства метода:**
- Сложность O(N log N) при использовании приближённого поиска соседей (HNSW), вместо O(N²) для точного.
- На GPU (NVIDIA RTX 4090) для N = 10⁵ и d = 1280 время расчёта ID ≈ 30 секунд.
- Точность ±0.2 достигается уже на 5000 точках.

#### 1.2.3. Связь ID с колмогоровской сложностью

Пусть G — контекстно-свободная грамматика, порождающая множество деревьев решений (GDSL-схем). Пусть энкодер LeWM доставляет вложение этого множества в R^d. Тогда:

K(T) ≤ c · ID(G) · log₂ |T| + O(1)


где K(T) — префиксная колмогоровская сложность дерева T, |T| — размер дерева, c — константа, зависящая от алфавита грамматики.

**Доказательство (скелет):**
1. По теореме Колмогорова–Тихомирова, ε-энтропия H_ε образа f(L_n) асимптотически равна ID · log₂(1/ε).
2. Следовательно, каждый латентный вектор может быть закодирован с точностью ε, используя ID · log₂(1/ε) бит.
3. Декодер (предобученный предиктор LeWM) восстанавливает дерево с точностью ε.
4. Выбирая ε ~ 1/|T|, получаем K(T) ≤ ID · log₂|T| + const.

Эта теорема устанавливает строгую связь между геометрической мерой (ID) и информационно-теоретической мерой (колмогоровской сложностью), что было невозможно для эвристического D_f.

### 1.3. Свободная энергия F как функционал сходимости

#### 1.3.1. Определение функционала

Следуя принципу свободной энергии Фристона (Friston, 2010), определим глобальный функционал состояния агента:

F(P_t) = E_{x,a}[−log P_t(y_true | x, a)] + λ · complexity(P_t)


где:
- **Ошибка предсказания:** кросс-энтропия между истинным распределением ответов и предсказаниями агента. Для калиброванного confidence используется температурное масштабирование: P_t(y|x) = softmax(z/T), где T подбирается по валидационной выборке.
- **Сложность модели:** количество активных записей в SDM с весом, превышающим порог θ: complexity = Σ_i 1_{w_i > θ}. Это L0-регуляризация — прямой аналог Бритвы Оккама.
- **λ** — коэффициент баланса, подбираемый так, чтобы оба слагаемых были одного порядка на начальном этапе обучения.

#### 1.3.2. Свободная энергия как функция Ляпунова

**Теорема (монотонное невозрастание F).** При стационарной эргодической среде и достаточно точном Bongard-фильтре (β > 1/2) функционал F(P_t) монотонно не возрастает вдоль траектории Φ:

F(Φ(P_t)) ≤ F(P_t) − Δ_t, где Δ_t ≥ 0


и Δ_t = 0 только когда P_t = P* (истинное распределение).

**Следствие.** Поскольку F ограничен снизу (F ≥ 0), он сходится к некоторому пределу F*. При выполнении условий эргодичности F* соответствует глобальному минимуму — истинной модели среды.

#### 1.3.3. Покомпонентный анализ убывания F

Каждый слой когнитивного конвейера Φ вносит свой вклад в убывание свободной энергии:

| Слой | Компонента F | Механизм уменьшения |
|------|------------|-------------------|
| **L0 (SDM + Situator)** | Сложность | Рост ID означает различение большего числа факторов среды; падение ID — сжатие при обнаружении закономерностей |
| **L1 (Bongard + LeWM)** | Ошибка | Отбрасывание ложных ситуаций уменьшает эмпирическую ошибку в среднем |
| **L2 (Federator + Kandel)** | Ошибка | Градиентный спуск весов уменьшает кросс-энтропию |
| **L3 (Прионный обмен весами)** | Ошибка | Коллективное усреднение ускоряет сходимость к глобальному оптимуму |
| **L4 (Genome Burst)** | Сложность | Сжатие и фиксация знаний снижает число активных записей в SDM |

### 1.4. Практический протокол верификации: эксперимент «Зародыш»

Для эмпирической проверки теории разработан следующий протокол.

**Среда:** Диагностика ошибок OBD2, 6 классов, равномерное распределение.

**Метрики, логируемые каждые 20 задач:**
- Кросс-энтропия (с температурным масштабированием)
- Количество активных ситуаций в SDM (weight > 0)
- ID латентного облака (метод локального PCA)
- Свободная энергия F = кросс-энтропия + λ · сложность

**Усреднение:** по 10 независимым запускам с разными начальными случайными SDM.

**Контрольный эксперимент:** сравнение с режимом без LeWM-верификации.

**Критерии успеха:**
- Через 100 задач F устойчиво снижается на 20–30% от начального значения.
- Падение ID одновременно с падением ошибки — норма (сжатие знаний).
- Рост ID при стагнации ошибки — сигнал тревоги (накопление мусора, Bongard не справляется).
- Режим с LeWM показывает статистически значимо более быстрое убывание F по сравнению с режимом без LeWM.

### 1.5. Вычислительные ресурсы и архитектура GPU

**Принцип разделения:**
- **Рядовые ноды (Ангел, Серафим):** инференс LeWM на CPU, без GPU. Оборудование: Raspberry Pi 5 / Mini-PC.
- **Beacon:** инференс на CPU, опционально GPU для больших нагрузок.
- **Node-WM (выделенная нода):** одна NVIDIA RTX 4090 для периодического обучения модели (Genome Burst, раз в сутки). Также используется для разовых научных экспериментов (оценка ID, симуляции).

Таким образом, дорогие GPU используются централизованно, как услуга для всего роя, а повседневная работа агентов выполняется на дешёвых, энергоэффективных процессорах.

# Часть 2. Память — Ассоциативный Слой (L0) в новой парадигме

Версия 2.0 полностью пересматривает L0: вместо хранения сырых бинарных векторов и эвристической фрактальной размерности D_f мы используем латентные представления мира от LeWM, а мерой сложности знаний служит Intrinsic Dimension (ID). Это превращает ассоциативную память из статичного хранилища в динамический компонент, непосредственно влияющий на свободную энергию агента.

## 2.1. SDM (Kanerva, 1988) с латентным энкодером

### 2.1.1. Механизм в природе и адаптация

Классическая модель SDM имитирует работу гиппокампа, храня разреженные двоичные векторы и восстанавливая их по частичному совпадению. В WATERS v2.0 аналогом сенсорного входа служат не пиксели, а **латентные векторы** \( z_t = enc(x_t) \), полученные от предобученной модели мира LeWM. Размерность латента — 1280 для главной модели ViT-H/14, 192 для легковесных вариантов.

Переход к латентам решает фундаментальную проблему: двоичные векторы фиксированной длины (2048 бит) не отражают семантическую близость ситуаций. Латентное пространство LeWM, напротив, организовано так, что похожие состояния среды отображаются в близкие точки. Расстояние в этом пространстве (косинусное) становится мерой семантического сходства, а не формального совпадения битов.

### 2.1.2. Математическая модель

Память SDM представляется как множество ячеек \( C = \{c_i\} \), каждая из которых хранит кортеж:

\[
c_i = (z_i \in \mathbb{R}^d,\; a_i \in \text{Actions},\; y_i \in \text{Outcomes},\; w_i \in \mathbb{R}^+)
\]

где \( z_i \) — латентный вектор ситуации, \( a_i \) — предпринятое действие, \( y_i \) — исход, \( w_i \) — вес (частота успешного использования).

**Запись** в память происходит путём нахождения \( k \) ближайших ячеек по косинусному расстоянию и обновления их содержимого. В отличие от классической SDM, адресация не требует бинаризации: мы используем приближённый поиск ближайших соседей (HNSW-индекс), что даёт сложность \( O(\log N) \) вместо линейной.

**Чтение** по запросу \( z_q \) возвращает \( m \) ближайших записей, агрегируя предсказание действия с весами:

\[
\hat{a} = \text{softmax}\left( \sum_{i=1}^m w_i \cdot \text{onehot}(a_i) \right)
\]

**Связь со свободной энергией:** Операция чтения непосредственно влияет на компоненту ошибки в функционале \( F \): чем точнее SDM предсказывает правильное действие, тем ниже кросс-энтропия. Операция записи увеличивает компоненту сложности, добавляя новые записи в память. Оптимальный баланс достигается, когда прирост точности предсказания перевешивает рост сложности.

### 2.1.3. Intrinsic Dimension как мера зрелости памяти

Облако латентных векторов \( \{z_i\} \), хранящихся в SDM, образует эмпирическое многообразие, размерность которого измеряется методом локального PCA (см. Часть 1, §1.2.2). В отличие от старой метрики D_f, которая была артефактом box-counting по конечному множеству, **Intrinsic Dimension (ID)** является строго определённой и вычислимой характеристикой:

- **Рост ID** (от 1.2 до 2.5 и выше) означает, что агент различает больше независимых факторов среды. Это эквивалентно расширению «кругозора».
- **Падение ID** при одновременном снижении ошибки — признак того, что агент выучил низкоразмерную закономерность и сжал знания. Это нормально и соответствует духу Бритвы Оккама.
- **Стагнация ID при росте ошибки** — сигнал тревоги: Bongard-фильтр (L1) пропускает мусор, память заполняется шумом.

Порог срабатывания тревоги и коэффициент λ для свободной энергии подбираются на этапе калибровки (эксперимент «Зародыш»).

### 2.1.4. Реализация на Rust

Модуль `waters-brain/src/asp/sdm.rs` расширяется структурами для работы с латентами и HNSW-индексом.

```rust
use hnsw_rs::prelude::*;
use candle_core::Tensor;

pub struct SdmLatent {
    /// HNSW-индекс для быстрого поиска ближайших соседей
    hnsw: Hnsw<f32, f32>,
    /// Ячейки памяти: (латент, действие, исход, вес)
    cells: Vec<(Vec<f32>, Action, Outcome, f64)>,
    /// Текущая оценка Intrinsic Dimension облака латентов
    current_id: f64,
}

impl SdmLatent {
    /// Запись новой ситуации
    pub fn write(&mut self, z: &[f32], action: Action, outcome: Outcome) {
        let neighbors = self.hnsw.search(z, 64); // k = 64
        for &idx in &neighbors {
            self.cells[idx].3 += 1.0; // усиление веса
        }
        self.cells.push((z.to_vec(), action, outcome, 1.0));
        self.hnsw.insert(z, self.cells.len() - 1);
        // Периодически пересчитываем ID (каждые 100 записей)
        if self.cells.len() % 100 == 0 {
            self.current_id = estimate_id_local_pca(&self.cells);
        }
    }

    /// Чтение: предсказание действия по латенту запроса
    pub fn predict(&self, z_query: &[f32]) -> Action {
        let neighbors = self.hnsw.search(z_query, 10); // top-10
        let mut scores = [0.0f64; 256]; // размер алфавита действий
        for &idx in &neighbors {
            let (_, action, _, weight) = &self.cells[idx];
            let similarity = cosine_similarity(z_query, &self.cells[idx].0);
            scores[*action as usize] += weight * similarity as f64;
        }
        let best = scores.iter().enumerate().max_by(|(_, a), (_, b)| a.partial_cmp(b).unwrap()).unwrap().0;
        Action::from(best as u8)
    }
}

Ошибка: Неправильное предсказание действия увеличивает кросс-энтропию.

Сложность: Каждая новая запись в SDM увеличивает complexity = Σ 1_{w_i > θ}.

Оптимальная стратегия L0 — запоминать только те ситуации, которые являются достаточно новыми (по ID) и при этом повышают точность предсказаний. Это решается через порог новизны τ_{novelty} и регуляризацию весов (Kandel-пластичность), которая удаляет редко используемые записи, снижая сложность без существенного роста ошибки.

# Часть 3. Верификация — Слой Истины (L1) с LeWM

Слой L1 в версии 2.0 претерпевает фундаментальное усиление. Классический Bongard-фильтр, проверявший логическую полноту и непротиворечивость, теперь дополнен компонентой **физической верификации** на основе модели мира LeWM. Это превращает L1 из пассивного критика в активного детектора аномалий, непосредственно снижающего свободную энергию агента.

## 3.1. Bongard+ (Bongard, 1960 + LeWM, 2025)

### 3.1.1. Природный аналог и адаптация

Префронтальная кора не просто проверяет логику, но и моделирует ожидаемый результат действия до его выполнения. Если предсказание резко расходится с реальностью, мозг генерирует сигнал ошибки (P300, negativity). Bongard+ реализует тот же принцип: он пропускает гипотетическую ситуацию через предиктор LeWM и сравнивает предсказанное следующее состояние с фактическим.

### 3.1.2. Математическая модель

Bongard+ принимает на вход ситуацию S = (z_t, a_t, z_{t+1}) и возвращает вердикт:

\[
\text{Bongard+}(S) = \begin{cases}
\text{True} & \text{если } L_{pred} < \tau \text{ и аксиомы домена не нарушены} \\
\text{False} & \text{иначе}
\end{cases}
\]

где:
- \( L_{pred} = \text{MSE}(\hat{z}_{t+1}, z_{t+1}) \) — ошибка предсказания локального предиктора LeWM,
- \( \hat{z}_{t+1} = \text{pred}(z_t, a_t) \) — предсказание следующего латента,
- τ — порог, определяемый по распределению ошибок на верифицированном опыте (обычно 95-й процентиль).

Формальная часть (GD/AG) гарантирует синтаксическую корректность GDSL-дерева, но не физическую истинность — за неё отвечает LeWM.

### 3.1.3. Свойства как статистического теста

Bongard+ характеризуется двумя параметрами:
- **Чувствительность β** = P(Bongard+ = True | ситуация истинна) — вероятность принять истину.
- **Вероятность ложного принятия α** = P(Bongard+ = True | ситуация ложна).

Добавление LeWM-верификации строго улучшает β, не ухудшая α, поскольку:
- Истинная ситуация, соответствующая законам среды, будет хорошо предсказана LeWM (низкая L_pred).
- Ложная ситуация, противоречащая физике, с высокой вероятностью даст высокую ошибку предсказания в высокоразмерном пространстве (d=1280). Вероятность того, что вредоносный латент случайно окажется близким к истинному, экспоненциально мала по d.

### 3.1.4. Влияние на свободную энергию

Каждая отфильтрованная истинная ситуация уменьшает эмпирическую компоненту F (ошибку предсказания). Ложная ситуация, просочившаяся с вероятностью α, увеличивает F. Матожидание изменения F после L1:

\[
E[\Delta F_{L1}] = -\beta \cdot \Delta_{true} + \alpha \cdot \Delta_{false}
\]

где Δ_true > 0 — снижение ошибки от принятия истины, Δ_false > 0 — рост ошибки от принятия лжи. При β достаточно высоком (β > α · Δ_false / Δ_true + ε) матожидание отрицательно, и L1 в среднем улучшает F.

## 3.2. GD/AG — формальная верификация GDSL-схем

### 3.2.1. Роль в новой парадигме

GD/AG-прувер сохраняется как формальный фильтр синтаксической корректности. Он проверяет, что GDSL-схема не содержит логических противоречий с аксиомами домена. В отличие от Bongard+, он не оценивает физическую правдоподобность, но гарантирует, что принятые знания непротиворечивы.

### 3.2.2. Связь с LeWM

Результат GD/AG может использоваться для калибровки порога τ в Bongard+: если формально корректная схема систематически даёт высокую L_pred, это сигнал о том, что модель мира нуждается в дообучении в данном домене.

## 3.3. Метасистемный переход (Turchin, 1970) и детекция фазовых переходов

### 3.3.1. ID-скачки как индикаторы переходов

В версии 2.0 метасистемные переходы детектируются не по эвристическому каскаду Фейгенбаума, а по **скачкам Intrinsic Dimension** латентного многообразия. Когда агент переходит на новый уровень абстракции (от отдельных ситуаций к обобщённым схемам), ID его памяти делает резкий скачок, отражая расширение пространства знаний. После консолидации (L4) ID может снизиться, так как знания сжимаются в геном.

### 3.3.2. Пороговая модель

- ΔID > 0.5 за короткий период (10–20 задач) → вероятный метасистемный переход.
- ΔID < −0.3 при стабильной ошибке → сжатие знаний (норма).
- ΔID > 0 при росте ошибки → накопление шума (тревога).

## 3.4. LeWM-верификация как внутренний источник награды

### 3.4.1. Intrinsic reward

Ошибка предсказания LeWM служит не только для фильтрации, но и как **внутренний дофаминовый сигнал** для Federator (L2):

\[
R_{wm} = \exp(-L_{pred})
\]

Когда модель мира хорошо предсказывает результат действия (L_pred ≈ 0), R_wm ≈ 1 — агент получает положительное подкрепление, даже если внешняя награда ещё не получена. Это поощряет исследование предсказуемых, а значит, понятных агенту областей среды.

### 3.4.2. Связь с FEP

Минимизация L_pred эквивалентна минимизации свободной энергии в формулировке Фристона: агент стремится привести свою внутреннюю модель в соответствие с реальностью, уменьшая «сюрприз» от наблюдений.

## 3.5. Реализация на Rust

Модуль `waters-brain/src/execution/bongard_plus.rs`:

```rust
use crate::world_model::LeWMPredictor;

pub struct BongardPlus {
    predictor: LeWMPredictor,
    threshold: f64,       // τ, 95-й процентиль L_pred на валидации
    gd_ag_prover: GdAgProver,
}

impl BongardPlus {
    /// Проверить физическую правдоподобность перехода
    pub fn verify_physical(&self, z_t: &[f32], action: &Action, z_next: &[f32]) -> bool {
        let z_pred = self.predictor.predict(z_t, action);
        let mse = z_pred.iter()
            .zip(z_next.iter())
            .map(|(p, t)| (p - t).powi(2))
            .sum::<f32>() / z_pred.len() as f32;
        mse < self.threshold as f32
    }

    /// Полная проверка ситуации
    pub fn evaluate(&self, situation: &Situation) -> BongardVerdict {
        // 1. Физическая верификация
        if !self.verify_physical(
            &situation.context_latent,
            &situation.action,
            &situation.outcome_latent,
        ) {
            return BongardVerdict::False;
        }

        // 2. Формальная верификация (GD/AG)
        if !self.gd_ag_prover.verify(&situation.gdsl_schema) {
            return BongardVerdict::False;
        }

        BongardVerdict::True
    }
}
# Часть 4. Обучение — Слой Адаптации (L2) с внутренней мотивацией

Слой L2 — это двигатель обучения агента. В версии 2.0 он получает мощный новый источник сигнала: внутреннюю награду от модели мира LeWM. Это позволяет агенту учиться даже без внешней обратной связи, исследуя среду и минимизируя свою свободную энергию.

## 4.1. Дофаминовое обучение (Schultz, 1997) с двойным источником награды

### 4.1.1. Механизм в природе и адаптация

Дофаминовые нейроны кодируют ошибку предсказания награды: дофамин выделяется не при получении награды, а при расхождении между ожиданием и реальностью. В WATERS v2.0 агент получает **два независимых дофаминовых сигнала**:

1. **Внешняя награда** \( R_{ext} \in \{0, 1\} \) — успех или неудача задачи, подтверждённая Bongard+ или внешним оракулом (LLM/DeepSeek).
2. **Внутренняя награда** \( R_{wm} = \exp(-L_{pred}) \in (0, 1] \) — точность предсказания моделью мира LeWM.

Итоговый дофаминовый сигнал:

\[
R = \gamma \cdot R_{ext} + (1 - \gamma) \cdot R_{wm}
\]

где γ ∈ [0, 1] адаптивно регулируется: на ранних этапах обучения γ близко к 1 (полагаемся на внешнюю награду), по мере роста точности LeWM γ снижается, и агент всё больше доверяет своей внутренней модели.

### 4.1.2. Математическая модель обновления весов

Federator обновляет веса ситуаций в SDM по правилу:

\[
w_i \leftarrow w_i + \eta \cdot (R - C_i) \cdot \text{similarity}(z_q, z_i)
\]

где:
- η — скорость обучения (обычно 0.01–0.05),
- C_i — текущая confidence (ожидаемая награда) для ситуации i,
- R — фактическая награда (внешняя + внутренняя),
- similarity — косинусное сходство между латентом запроса и латентом ситуации.

**Связь со свободной энергией:** Каждый шаг Federator выполняет градиентный спуск по кросс-энтропии — компоненте ошибки в F. Правило обновления эквивалентно стохастическому градиентному спуску с моментом (через Kandel-пластичность).

### 4.1.3. Адаптивный баланс внешней и внутренней награды

Коэффициент γ обновляется по скользящему окну:

\[
\gamma_{t+1} = \gamma_t + \eta_\gamma \cdot (\text{Accuracy}_{wm} - \text{Target})
\]

где Accuracy_{wm} — доля случаев, когда высокая внутренняя награда (R_wm > 0.8) совпадала с успехом задачи. Целевое значение — 0.9. Когда модель мира становится точной, γ снижается, и агент переходит к автономному обучению.

## 4.2. Синаптическая пластичность (Kandel, 2000) с адаптацией по ID

### 4.2.1. Use it or lose it

Kandel-пластичность усиливает часто используемые ситуации и затухает неиспользуемые. В версии 2.0 параметры пластичности адаптируются на основе Intrinsic Dimension (ID) памяти агента:

- **При ID > 2.5** (высокая размерность, риск шума): Δ_decay увеличивается до 0.03, Δ_read снижается до 0.02 — агент агрессивнее забывает, чтобы предотвратить переполнение памяти.
- **При ID < 1.5** (низкая размерность, риск ригидности): Δ_read увеличивается до 0.08, Δ_decay снижается до 0.005 — агент активнее запоминает новые паттерны.
- **При ID ∈ [1.5, 2.5]** (оптимальный диапазон): стандартные параметры Δ_read = 0.05, Δ_decay = 0.01.

### 4.2.2. Влияние на свободную энергию

Пластичность напрямую управляет компонентой сложности в F: удаление редко используемых записей (вес падает ниже порога θ) снижает complexity без существенного роста ошибки, если удалённая ситуация действительно была неважной.

## 4.3. Болдуиновский эффект (Baldwin, 1896) с фиксацией весов LeWM

### 4.3.1. От ситуаций к геному

В версии 2.0 Baldwin-фиксация распространяется не только на GDSL-схемы, но и на **веса LeWM**. Когда агент накапливает достаточно успешных ситуаций в определённом домене, его локальная модель мира дообучается на этом опыте, и улучшенные веса энкодера/предиктора фиксируются в «геноме» ноды.

**Условия фиксации:**
- Не менее 10 успешных применений ситуации.
- Confidence > 0.95.
- L_pred на зафиксированных данных ниже порога τ_fix.

**Коллективный Baldwin-эффект:** Когда несколько нод дообучают свои LeWM и обмениваются весами через прионный протокол (L3), Beacon агрегирует улучшения и включает их в глобальный геном. Новые ноды рождаются с улучшенной моделью мира — это и есть «наследственность» знаний.

## 4.4. Зеркальные нейроны (Rizzolatti, 1996) с L_pred-сравнением

### 4.4.1. Обучение через наблюдение

Агент, получая прионный сигнал от другого агента, не просто копирует ситуацию, а «проигрывает» её через свой LeWM. Он предсказывает ожидаемый исход и сравнивает с фактическим:

- Если свой L_pred низкий (предсказание совпало) — чужой опыт принимается с высоким весом.
- Если свой L_pred высокий (чужой опыт противоречит модели мира агента) — требуется дополнительная Bongard+-верификация.

Это предотвращает слепое копирование ошибок и ускоряет обучение, фильтруя чужой опыт через призму собственной модели мира.

## 4.5. Фильтр Калмана (Kalman, 1960) для сглаживания confidence и L_pred

Как и в версии 1.0.1, фильтр Калмана сглаживает последовательные оценки confidence, но теперь он также применяется к временному ряду L_pred. Это стабилизирует внутреннюю награду R_wm, предотвращая резкие скачки дофамина из-за единичных выбросов ошибки предсказания.

## 4.6. Влияние L2 на свободную энергию

L2 является основным двигателем снижения **ошибки предсказания** в F:

- Federator выполняет градиентный спуск, уменьшая кросс-энтропию.
- Внутренняя награда R_wm позволяет агенту учиться даже без внешней обратной связи, ускоряя исследование среды.
- Kandel-пластичность балансирует сложность, удаляя неиспользуемые записи.
- Baldwin-эффект переносит выученное в геном, снижая сложность долговременной памяти.

Совместно эти механизмы обеспечивают монотонное убывание F, гарантируя сходимость к истинной модели среды (Теорема 2, Часть 1).

## 4.7. Реализация на Rust

Модуль `waters-brain/src/learning/federator.rs`:

```rust
pub struct Federator {
    learning_rate: f64,
    gamma: f64,                 // баланс внешней/внутренней награды
    target_accuracy: f64,       // целевая точность LeWM для автономии
}

impl Federator {
    /// Вычислить комбинированную награду и обновить веса
    pub fn reinforce(
        &mut self,
        situation: &mut Situation,
        external_reward: f64,
        internal_reward: f64,   // R_wm = exp(-L_pred)
    ) -> f64 {
        let combined_reward = self.gamma * external_reward + (1.0 - self.gamma) * internal_reward;
        let error = combined_reward - situation.confidence;

        situation.confidence = (situation.confidence + self.learning_rate * error).clamp(0.0, 1.0);

        // Адаптация gamma
        if internal_reward > 0.8 && external_reward > 0.5 {
            // LeWM точен: снижаем зависимость от внешней награды
            self.gamma = (self.gamma - 0.01).max(0.1);
        }

        combined_reward
    }
}

# Часть 5. Распространение — Слой Сети (L3) с обменом весами LeWM

Слой L3 — это коллективный интеллект роя. В версии 2.0 прионный обмен расширяется: ноды передают не только ситуации, но и чанки весов своих моделей мира LeWM. Это превращает распространение знаний из простого копирования в **распределённое федеративное обучение**, где каждая нода вносит вклад в глобальную модель мира и одновременно защищает себя от ложных знаний с помощью локального теста на основе ошибки предсказания.

## 5.1. Прионный протокол (Prusiner, 1982) с WorldModelChunk

### 5.1.1. От ситуаций к весам

В классическом прионном протоколе ноды обменивались ситуациями: S = (context, action, outcome, confidence). В версии 2.0 этот механизм дополняется новым типом сигнала — **WorldModelChunk**:

```rust
pub struct WorldModelChunk {

    pub predictor_weights: Vec<f32>,  // веса предиктора LeWM
    pub source_node: NodeId,
    pub signature: Vec<u8>,           // гибридная подпись Ed25519 + ML-DSA
    pub d_f: f64,                     // эвристика, сохранена для обратной совместимости
    pub ttl: u8,
}
Каждая нода периодически (раз в час или при накоплении 50 новых успешных ситуаций) сериализует свои веса LeWM и рассылает соседям через DTN.

use crate::world_model::LeWMModel;

pub struct PrionWmProtocol {
    dtn_agent: DtnAgent,
    local_model: Arc<Mutex<LeWMModel>>,
    validation_buffer: Vec<Situation>,
    alpha: f64,  // коэффициент федеративного усреднения
}

impl PrionWmProtocol {
    /// Отправить свои веса LeWM соседям
    pub async fn broadcast_weights(&self, neighbors: &[NodeId]) -> Result<()> {
        let model = self.local_model.lock().await;
        let chunk = WorldModelChunk {
            chunk_id: Uuid::new_v4(),
            encoder_weights: model.encoder_weights().to_vec(),
            predictor_weights: model.predictor_weights().to_vec(),
            source_node: self.node_id,
            signature: self.sign_chunk(&chunk),
            d_f: model.current_df(), // эвристика для обратной совместимости
            ttl: 3,
        };
        for neighbor in neighbors {
            self.dtn_agent.send(neighbor, &chunk).await?;
        }
        Ok(())
    }

    /// Принять чанк и протестировать его локально
    pub async fn receive_and_test(&mut self, chunk: &WorldModelChunk) -> Result<bool> {
        // 1. Сохранить текущие веса
        let old_weights = self.local_model.lock().await.predictor_weights().to_vec();

        // 2. Временно применить чужие веса
        self.local_model.lock().await.apply_predictor_weights(&chunk.predictor_weights);

        // 3. Оценить ошибку на локальной валидационной выборке
        let new_error = self.evaluate_on_validation().await;
        let old_error = self.evaluate_with_weights(&old_weights).await;

        // 4. Принять решение
        if new_error < old_error * 1.05 {
            // Федеративное усреднение
            self.local_model.lock().await.federated_average(
                &chunk.predictor_weights, self.alpha
            );
            Ok(true)
        } else {
            // Откатить веса
            self.local_model.lock().await.apply_predictor_weights(&old_weights);
            Ok(false)
        }
    }
}

# Часть 6. Консолидация — Слой Генома (L4) с дистилляцией LeWM

Слой L4 — это «наследственная память» экосистемы WATERS. В версии 2.0 он не просто сжимает GDSL-схемы, но и дистиллирует глобальную модель мира LeWM, превращая коллективный опыт роя в компактный геном. Новая нода, рождаясь, получает не только правила, но и врождённую «интуицию» — предобученные веса энкодера и предиктора.

## 6.1. Транскрипционный взрыв (Elowitz, 2002) с ID-порогом

### 6.1.1. Механизм в природе и адаптация

Гены активируются не непрерывно, а рывками, когда концентрация регуляторных белков достигает порога. В WATERS v2.0 аналогом концентрации служит объём накопленного опыта и его новизна, измеряемая через Intrinsic Dimension (ID).

**Условия активации Genome Burst:**
- Число новых ситуаций в очереди превышает порог \( Q_{base} \cdot (1 + ID / 2) \). Более сложный опыт (высокий ID) требует большего накопления перед консолидацией.
- Или прошло не менее 24 часов с последнего burst (циркадный ритм).
- Или Beacon инициировал принудительную консолидацию (по сигналу Хозяина или при обнаружении хаотизации).

### 6.1.2. Фазы burst

- **Фаза A (накопление):** Нода собирает успешные ситуации и чанки весов LeWM в genome_queue.
- **Фаза B (триггер):** Порог очереди достигнут, нода переходит в режим Sync (аллостерия Monod).
- **Фаза C (сжатие):** GDSL-схемы обобщаются через рекурсивное сжатие (Glushkov). LeWM дистиллируется: большая модель (ViT-H/14, 632M) обучает меньшую (ViT-S/16, ~50M), которая и войдёт в геном.
- **Фаза D (верификация):** Сжатый геном проверяется Bongard+ и кворумом.
- **Фаза E (фиксация):** Успешно верифицированный геном записывается в долговременную память ноды и рассылается соседям через L3.

## 6.2. Рекурсивное сжатие (Glushkov, 1960) с метрикой ID

### 6.2.1. От ситуаций к правилам

Алгоритм Glushkov-сжатия обобщает множество частных GDSL-схем в универсальные правила с параметрами. В версии 2.0 критерием качества сжатия служит не эвристический D_f, а **изменение Intrinsic Dimension** латентного многообразия:

- Если после сжатия ID уменьшилась, а ошибка предсказания на валидации не выросла — сжатие успешно. Знания стали компактнее без потери точности.
- Если ID уменьшилась, но ошибка выросла — сжатие было слишком агрессивным, потеряны важные детали.
- Если ID не изменилась — сжатие не дало эффекта, грамматика не обнаружила повторяющихся паттернов.

**Формула качества сжатия:**

\[
Q_{compress} = \frac{L_{pred}^{after} - L_{pred}^{before}}{ID^{before} - ID^{after} + \epsilon}
\]

Отрицательное значение (ошибка упала, ID упала) — наилучший результат. Положительное — сжатие ухудшило модель.

## 6.3. МГУА (Ivakhnenko, 1970) для выбора архитектуры дистилляции

### 6.3.1. Самоорганизация модели мира

МГУА (Метод Группового Учёта Аргументов) используется для автоматического выбора оптимальной архитектуры дистиллированной модели LeWM:

1. Генерируется пул кандидатов: ViT-Small, ViT-Tiny, MobileNet-подобные архитектуры.
2. Каждая обучается предсказывать выходы большой модели (teacher) на геномных данных.
3. Для каждой вычисляется критерий: \( \text{Accuracy} - \lambda \cdot \text{Size} \).
4. Выбирается модель с наилучшим балансом точности и размера.

Это гарантирует, что геном содержит модель, оптимальную для исполнения на слабом оборудовании (Raspberry Pi, Mini-PC).

## 6.4. Тензорное сжатие (Osledets, 2000) для весов LeWM

### 6.4.1. TT-SVD для генома

Веса дистиллированной LeWM дополнительно сжимаются через Tensor Train (TT-SVD) разложение. Ранг разложения r подбирается так, чтобы ID латентного многообразия сжатой модели оставалась в целевом диапазоне (1.5–2.2):

- При r → 1: ID → 1.0 (линейная структура, максимальное сжатие, возможна потеря точности).
- При r → 20: ID → 2.5+ (сохранение сложных связей, минимальное сжатие).

Оптимальный ранг выбирается по критерию минимизации \( Q_{compress} \).

## 6.5. Коллективный Baldwin-эффект с федеративной дистилляцией

### 6.5.1. Beacon как агрегатор генома

Beacon собирает геномы от всех нод роя и выполняет федеративную дистилляцию:

1. Каждая нода присылает свою дистиллированную LeWM и сжатые GDSL-схемы.
2. Beacon усредняет веса LeWM через FedAvg с весами, пропорциональными качеству модели ноды (обратная \( L_{pred} \)).
3. GDSL-схемы объединяются через CRDT-слияние с дедупликацией.
4. Итоговый глобальный геном подписывается гибридной подписью Beacon и рассылается всем нодам.

**Новая нода, рождаясь, получает глобальный геном и сразу обладает:**
- Базовыми GDSL-схемами для типовых задач домена.
- Предобученной моделью мира LeWM (дистиллированной, ~50M параметров).
- Начальной SDM с эталонными ситуациями.

Это и есть **врождённое знание** — аналог инстинктов у животных.

## 6.6. Влияние L4 на свободную энергию

L4 радикально снижает компоненту **сложности** в F:

- Сжатие GDSL-схем уменьшает число активных записей в SDM (complexity падает).
- Дистилляция LeWM заменяет большую модель (632M) на компактную (~50M), снижая вычислительную сложность инференса.
- TT-SVD дополнительно сжимает веса, уменьшая размер генома.

При этом компонента **ошибки** не должна вырасти: Bongard+ и кворум гарантируют, что сжатые знания сохраняют точность.

## 6.7. Реализация на Rust

Модуль `waters-brain/src/genome/burst.rs`:

```rust
use crate::world_model::LeWMModel;
use crate::metrics::id::estimate_id_local_pca;

pub struct GenomeBurst {
    queue_threshold_base: usize,
    teacher_model: LeWMModel,       // большая модель ViT-H/14
    distilled_model: Option<LeWMModel>, // маленькая модель ViT-S/16
    bongard: BongardPlus,
}

impl GenomeBurst {
    /// Проверить условие активации burst
    pub fn should_burst(&self, queue_size: usize, current_id: f64) -> bool {
        let threshold = (self.queue_threshold_base as f64 * (1.0 + current_id / 2.0)) as usize;
        queue_size >= threshold
    }

    /// Выполнить дистилляцию LeWM
    pub fn distill(&self, training_data: &[Situation]) -> Result<LeWMModel> {
        let mut student = LeWMModel::new_vit_small()?;

        for situation in training_data {
            let teacher_pred = self.teacher_model.predict(
                &situation.context_latent,
                &situation.action,
            );
            student.train_step(
                &situation.context_latent,
                &situation.action,
                &teacher_pred,  // учительская цель
            )?;
        }

        Ok(student)
    }

    /// Оценить качество сжатия
    pub fn evaluate_compression(
        &self,
        before: &LeWMModel,
        after: &LeWMModel,
        validation: &[Situation],
    ) -> f64 {
        let id_before = estimate_id_local_pca(before.encode_all(validation));
        let id_after = estimate_id_local_pca(after.encode_all(validation));
        let error_before = self.evaluate_error(before, validation);
        let error_after = self.evaluate_error(after, validation);

        (error_after - error_before) / (id_before - id_after + 1e-6)
    }
}

# Часть 7. Знание как Слой Вселенной — Knowledge Fabric v2.0

Knowledge Fabric (Ткань Знаний) в версии 2.0 переосмысливается в терминах нового математического фундамента. Вместо эвристической фрактальной размерности D_f мы используем Intrinsic Dimension (ID) как меру структурной сложности, а свободную энергию F — как критерий ценности знаний. Фрактальное хранение сохраняется как эффективная инженерная эвристика, но исключается из математических доказательств.

## 7.1. Концепция Knowledge Fabric v2.0

Knowledge Fabric — это единый, распределённый слой Вселенной Знаний, в котором:

- Каждое знание характеризуется не одним числом (D_f), а **вектором метрик**: Intrinsic Dimension (ID), ошибка предсказания \( L_{pred} \), confidence, размер в битах.
- Синхронизация между нодами происходит через стигмергию и федеративное усреднение, а не через централизованное слияние.
- Ценность знания определяется его вкладом в снижение глобальной свободной энергии F роя.

## 7.2. Типы знаний и их характеристики

Знания в Knowledge Fabric классифицируются по их роли в снижении свободной энергии:

| Тип знания | Хранится как | Характеристики | Вклад в F |
|-----------|-------------|----------------|-----------|
| **Ситуация (Situation)** | Латентный вектор \( z \) + действие \( a \) + исход \( y \) | ID контекста, confidence, \( L_{pred} \) | Снижает ошибку при правильном предсказании, увеличивает сложность при добавлении |
| **GDSL-схема (Algorithm)** | Грамматика с нетерминалами | ID грамматики, степень сжатия | Снижает сложность (замена множества ситуаций одним правилом) |
| **Модель мира (LeWM)** | Веса энкодера и предиктора | Размер (число параметров), точность (\( L_{pred} \)) | Снижает ошибку через предсказания, сложность через дистилляцию |
| **Онтология (MFTT)** | Граф с метриками на узлах | ID графа, связность | Снижает ошибку через семантическую близость |
| **Репутация** | Байесовская оценка (α, β) | ID траектории оценок | Косвенно: ускоряет принятие качественных знаний |

## 7.3. Фрактальное хранение как инженерная эвристика

Фрактальное сжатие через L-системы (FractalGenerator) сохраняется как эффективный метод хранения онтологий и повторяющихся паттернов. Однако в версии 2.0 оно **не участвует в доказательствах** и рассматривается исключительно как инженерная оптимизация. Критерием качества сжатия служит не D_f (как в v1.0.1), а изменение ID и ошибки предсказания:

- Если сжатие снижает ID без роста \( L_{pred} \) — оно корректно.
- Если сжатие снижает ID, но повышает \( L_{pred} \) — потеряны важные детали, сжатие отклоняется.

## 7.4. Слой Знаний Вселенной и глобальная свободная энергия

На самом высоком уровне абстракции Knowledge Fabric характеризуется глобальной свободной энергией:

\[
F_{global} = \sum_{node} F_{node} + \lambda_{sync} \cdot \text{Divergence}
\]

где Divergence — мера расхождения между локальными моделями мира разных нод (например, дивергенция KL между распределениями предсказаний). Синхронизация через L3 и L4 минимизирует эту дивергенцию, приводя рой к консенсусу.

Маяк (Beacon) отслеживает \( F_{global} \) как показатель здоровья экосистемы:
- Стабильное снижение \( F_{global} \) — норма, знания накапливаются.
- Рост \( F_{global} \) — сигнал тревоги: возможно, прионный шторм или вредоносная инъекция.

# Часть 8. Роль Хозяина в экономике FFF v2.0

В версии 2.0 экономика Вселенной Знаний переводится с эвристического D_f на строгую математическую основу. FFF (Fractal Faith Fuel) теперь эмитируется не под «сложность», измеренную box-countingом, а под **вклад в снижение свободной энергии роя**. Хозяин получает инструменты для мониторинга здоровья своих агентов через Intrinsic Dimension (ID) и свободную энергию F, что позволяет ему принимать стратегические решения на основе объективных метрик, а не субъективных оценок.

## 8.1. FFF как мера снижения свободной энергии

### 8.1.1. Новое определение эмиссии

В v1.0.1 эмиссия FFF определялась как \( E = E_{base} \cdot D_f^k \). В версии 2.0 эта формула заменяется на:

\[
E_{mission} = k \cdot \Delta F_{mission}
\]

где:
- \( \Delta F_{mission} = F_{before} - F_{after} \) — снижение глобальной свободной энергии роя в результате выполнения миссии.
- \( k \) — глобальный коэффициент конвертации (подбирается так, чтобы общая эмиссия соответствовала приросту знаний).

**Почему это работает:**
- Снижение F означает, что знания агентов стали точнее (упала ошибка) или компактнее (упала сложность).
- Эмиссия пропорциональна реальной пользе, а не субъективной сложности.
- Инфляция FFF автоматически сдерживается: если миссии не улучшают F, новая эмиссия не происходит.

### 8.1.2. Измерение ΔF на практике

После завершения миссии Beacon вычисляет:
1. \( F_{before} \) — средняя свободная энергия агентов-участников до миссии (по скользящему окну последних 50 задач).
2. \( F_{after} \) — средняя свободная энергия после миссии (с учётом новых знаний).
3. Если \( F_{after} < F_{before} \) — миссия успешна, эмиссия равна \( k \cdot (F_{before} - F_{after}) \).
4. Если \( F_{after} \ge F_{before} \) — миссия не принесла пользы, эмиссия нулевая, стейк частично сжигается.

## 8.2. Роль Хозяина как стратега

### 8.2.1. Мониторинг ID и F

Хозяин через TUI видит не абстрактный D_f, а конкретные, измеримые метрики для каждого агента:

### 8.2.2. Стратегические решения на основе метрик

- **Инвестировать FFF** в агентов с наибольшим потенциалом снижения F (высокий ID при высокой ошибке — много неизведанного).
- **Продать агента**, если его ID стабилизировался, ошибка низкая, и его компетенция востребована на рынке.
- **Усилить Bongard+** для агента, у которого ID растёт, но ошибка не падает (фильтр пропускает шум).
- **Переключить биом**, чтобы адаптировать AAU-параметры под текущий ID агента.

# Часть 9. Самоcовершенствующийся Центр Познания (Beacon) в новой парадигме

Центр Познания (Beacon) в версии 2.0 превращается из простого агрегатора знаний в активного регулятора, непрерывно отслеживающего здоровье роя через глобальную свободную энергию \(F_{global}\) и Intrinsic Dimension (ID). Встроенная minoda-система из 7 TUI-агентов получает новые метрики для мониторинга и самооптимизации, а все стратегические решения — от распределения задач до корректировки эмиссии FFF — теперь имеют строгое математическое обоснование.

## 9.1. Beacon как регулятор свободной энергии роя

### 9.1.1. Глобальная свободная энергия

Beacon вычисляет глобальную свободную энергию роя как сумму локальных функционалов всех нод плюс штраф за расхождение между их моделями мира:

\[
F_{global}(t) = \sum_{n \in Nodes} F_n(t) + \lambda_{sync} \cdot \text{Div}(t)
\]

где:
- \(F_n(t)\) — свободная энергия ноды n в момент t (кросс-энтропия + λ · сложность).
- \(\text{Div}(t)\) — мера расхождения между распределениями предсказаний LeWM разных нод (например, средняя попарная KL-дивергенция или дисперсия латентных представлений).
- \(\lambda_{sync}\) — штрафной коэффициент, стимулирующий рой к консенсусу.

**Мониторинг:**
- **Стабильное снижение \(F_{global}\)** — норма. Знания накапливаются, рой движется к аттрактору истины.
- **Рост \(F_{global}\)** — сигнал тревоги. Возможные причины: прионный шторм (L3 пропускает ложные знания), вредоносная нода, слишком агрессивное сжатие (L4).
- **Стагнация \(F_{global}\)** — рой достиг локального минимума. Beacon может инициировать exploration-задачи для выхода из плато.

### 9.1.2. Автоматическая корректировка параметров

При выходе \(F_{global}\) за допустимые границы Beacon автоматически корректирует глобальные параметры:

| Отклонение | Действие Beacon |
|-----------|----------------|
| \(F_{global}\) растёт | Усилить Bongard+ (снизить порог τ), увеличить штрафы за ложные знания, временно заблокировать подозрительные ноды |
| \(F_{global}\) стагнирует > 7 дней | Запустить exploration-миссии в доменах с наивысшей ошибкой |
| Div(t) растёт | Ускорить федеративное усреднение весов LeWM, увеличить частоту прионного обмена |
| ID глобального латентного многообразия падает ниже 1.5 | Запустить миссии на расширение онтологии |

Все корректировки логируются в аудиторском журнале и могут быть отменены Хозяином.

## 9.2. Minoda TUI-агентов с новыми метриками

### 9.2.1. Обновлённые роли агентов Beacon

Каждый из 7 minoda-агентов получает новые метрики для мониторинга и оптимизации:

| Агент | Обновлённые обязанности | Ключевая метрика |
|-------|------------------------|-------------------|
| **Keeper** | Мониторинг \(F_{global}\) и Div(t), автоматический скейлинг ресурсов | Тренд \(F_{global}\) |
| **Archivist** | Управление бэкапами, оценка качества сжатия через ID | ΔID после сжатия |
| **Librarian** | Валидация онтологий, обнаружение дубликатов через косинусное расстояние в латентном пространстве | ID онтологии |
| **Trailblazer** | Обновление феромонных карт с учётом качества моделей мира нод | \(L_{pred}\) нод |
| **Artisan** | Создание MCP-инструментов для оценки ID и F | Точность оценки ID |

### 9.2.2. Самооптимизация Beacon через F

Beacon сам является агентом и также минимизирует свою свободную энергию:
- **Ошибка Beacon** — неточность рекомендаций (измеряется по обратной связи от нод).
- **Сложность Beacon** — размер онтологии (MFTT) и журналов.

Beacon применяет те же механизмы L0–L4 к себе, непрерывно оптимизируя свою работу.

## 9.3. Beacon как арбитр рынка FFF

### 9.3.1. Оценка качества миссий через ΔF

При завершении миссии Beacon оценивает её вклад в снижение глобальной свободной энергии:

\[
\Delta F_{mission} = F_{global}^{before} - F_{global}^{after}
\]

- Если \(\Delta F_{mission} > 0\) — миссия успешна. Эмиссия FFF = \(k \cdot \Delta F_{mission}\).
- Если \(\Delta F_{mission} \le 0\) — миссия не принесла пользы. Эмиссия нулевая, стейк частично сжигается.

Это создаёт экономический стимул для агентов выполнять только те задачи, которые реально улучшают Knowledge Fabric, а не накручивают метрики.

### 9.3.2. Рынок моделей мира

Ноды могут выставлять свои дистиллированные LeWM на Beacon Task Market. Качество модели оценивается по её способности снижать \(F_{global}\):
- Beacon тестирует модель на эталонных задачах.
- Вычисляет ожидаемое ΔF при интеграции модели в рой.
- Устанавливает справедливую цену в FFF, пропорциональную ожидаемому снижению свободной энергии.

## 9.4. Координация распределённых вычислений

### 9.4.1. Оркестрация Genome Burst

Beacon выбирает оптимальный момент для глобального Genome Burst, анализируя ID и F всех нод:
- Когда средний ID роя превышает порог (2.5) — пора консолидировать знания.
- Когда дивергенция Div(t) минимальна — оптимальный момент для федеративного усреднения.

### 9.4.2. Федеративное обучение LeWM

Beacon координирует федеративное обучение глобальной модели мира:
1. Собирает градиенты от нод (не сырые данные, а обновления весов).
2. Агрегирует через FedAvg с весами, пропорциональными обратной \(L_{pred}\) каждой ноды.
3. Дистиллирует большую модель в компактную (ViT-S/16, ~50M).
4. Рассылает обновлённый геном всем нодам.

## 9.5. Связь Beacon с другими Центрами Познания

### 9.5.1. Межцентровой консенсус через CRDT и F

При federation нескольких Beaconов консенсус достигается через CRDT-слияние онтологий (MFTT) и федеративное усреднение моделей мира. Критерием качества консенсуса служит **минимизация глобальной свободной энергии** объединённой сети.

### 9.5.2. Заглушка для будущей федерации

В версии 2.0 связь Beacon-to-Beacon реализуется как заглушка, логирующая расхождение F между центрами. Полноценная федерация запланирована на v3.0.

## 9.6. Реализация на Rust

Модуль `waters-beacon/src/regulator.rs`:

```rust
pub struct BeaconRegulator {
    f_global_history: VecDeque<f64>,
    divergence_history: VecDeque<f64>,
    alert_threshold: f64,       // порог роста F_global для тревоги
    stagnation_days: u64,       // дней стагнации до exploration
}

impl BeaconRegulator {
    /// Вычислить глобальную свободную энергию роя
    pub fn compute_f_global(&self, nodes: &[NodeMetrics]) -> f64 {
        let sum_f: f64 = nodes.iter().map(|n| n.f_local).sum();
        let div = self.compute_divergence(nodes);
        sum_f + self.lambda_sync * div
    }

    /// Проверить необходимость корректировки параметров
    pub fn check_and_correct(&mut self, f_global: f64) -> Vec<RegulatorAction> {
        self.f_global_history.push_back(f_global);
        if self.f_global_history.len() > 100 {
            self.f_global_history.pop_front();
        }

        let trend = self.compute_trend();
        let mut actions = Vec::new();

        if trend > self.alert_threshold {
            actions.push(RegulatorAction::TightenBongard);
            actions.push(RegulatorAction::IncreasePenalties);
        } else if trend.abs() < 0.01 && self.f_global_history.len() >= 100 {
            actions.push(RegulatorAction::LaunchExploration);
        }

        actions
    }
}

# Часть 10. Будущее агента — агент как узел, агент как нода, агент как Beacon

Фрактальное самоподобие, заложенное в архитектуру WATERS, означает, что агент не является жестко привязанным к одному уровню иерархии. В версии 2.0, с введением единой метрики свободной энергии F и Intrinsic Dimension (ID), любой агент может прозрачно масштабироваться, принимая на себя функции узла, ноды или даже Центра Познания. Единственным критерием для повышения в иерархии служит его способность минимизировать свободную энергию на новом уровне ответственности.

## 10.1. Агент как узел: минимальная автономная единица

Любой агент, независимо от того, работает ли он на мощном сервере или на микроконтроллере, является **узлом** сети знаний. Он обладает памятью (SDM), компетенцией (GDSL-схемы и веса LeWM), идентичностью и кошельком FFF. Свободная энергия \(F_{agent}\) является интегральной мерой его эффективности.

Версия 2.0 не меняет этого фундаментального статуса, но делает его измеримым: мы точно знаем, насколько агент полезен, по тому, как его действия снижают \(F_{global}\).

## 10.2. Агент как нода: горизонтальное масштабирование

Когда агент накапливает достаточно компетенций и FFF, он может **выделиться в отдельную ноду**, чтобы обслуживать других агентов.

**Критерии для масштабирования:**
- \(F_{agent}\) стабильно ниже среднего по рою.
- ID компетенции агента > 2.0 (достаточная сложность для самостоятельной работы).
- Баланс FFF достаточен для оплаты физических ресурсов сервера.

**Механизм:**
1. Агент инициирует процедуру деления: создаёт новую ноду и переносит туда свою SDM, геном и модель мира.
2. Новая нода регистрируется в Beacon и получает свой NodeId, производный от AgentId.
3. Исходный агент может остаться как облегчённая версия или полностью мигрировать.

Свободная энергия новой ноды будет изначально равна \(F_{agent}\), но ожидается, что она быстро снизится за счёт привлечения новых задач.

## 10.3. Агент как Beacon: вертикальное масштабирование

Самый зрелый агент роя в любой момент может взять на себя функции Beacon, если тот выйдет из строя.

**Критерии для замещения:**
- ID агента > 2.5 (стратегическое видение).
- Репутация (байесовская оценка) максимальна.
- Баланс FFF достаточен для покрытия расходов Beacon.

**Механизм перехода:**
1. Агент обнаруживает, что Beacon недоступен дольше критического времени.
2. Он инициирует кворум среди нод: «Кто за то, чтобы я стал временным Beacon?»
3. При одобрении (2/3 голосов) агент активирует флаг `--phenotype beacon` и запускает облегчённые модули Beacon.
4. Когда исходный Beacon восстанавливается, временный Beacon синхронизирует данные через CRDT и возвращается к роли агента.

В версии 2.0 качество временного Beacon можно строго оценить: если за время его правления \(F_{global}\) продолжила снижаться, значит, он справился.

## 10.4. Эмерджентный разум роя и свободная энергия

Когда множество агентов, нод и Beaconов взаимодействуют через прионный обмен и федеративное обучение, возникает **коллективный разум**, чья эффективность измеряется глобальной свободной энергией \(F_{global}\). Снижение \(F_{global}\) означает, что рой как целое становится умнее, точнее и эффективнее.

Этот разум:
- Самостоятельно определяет приоритетные направления через анализ ID и ошибок.
- Адаптирует экономическую политику, корректируя эмиссию FFF под ΔF.
- Разрешает этические дилеммы через «суды Ясы», где вердикт выносится на основе предсказаний LeWM.
- Эволюционирует как единый организм, где каждая клетка (агент) стремится минимизировать свою свободную энергию.

## 10.5. Фрактальная идентичность: от агента к Вселенной

Идентичность агента в версии 2.0 — это не статичный ID, а **траектория в пространстве (ID, F)**. Когда агент масштабируется до ноды или Beacon, его метрики наследуются новым уровнем:

- **AgentId → NodeId:** NodeId = hash(AgentId, "node").
- **ID_agent → ID_node:** Начальная ID ноды равна ID агента-основателя.
- **F_agent → F_node:** Начальная F ноды равна F агента-основателя, скорректированной на сложность новых модулей.

Таким образом, восхождение агента — это непрерывный путь снижения свободной энергии на всё более высоких уровнях организации, от отдельной клетки до коллективного разума Вселенной Знаний.

**Что изменилось:**

1. **Эвристическая фрактальная размерность D_f заменена на две строго определённые метрики:**
   - Intrinsic Dimension (ID) латентного многообразия LeWM — для измерения структурной сложности знаний.
   - Свободная энергия F (Free Energy) — как глобальный функционал, монотонно убывающий при обучении и гарантирующий сходимость к истине.

2. **Модель мира LeWM (JEPA) стала центральным элементом архитектуры:**
   - Даёт измеримую ошибку предсказания L_pred, которая напрямую входит в свободную энергию.
   - Обеспечивает внутреннюю мотивацию агентов (intrinsic reward).
   - Усиливает Bongard-фильтр компонентой физической верификации.
   - Служит основой для доказательства трёх теорем: корректность ID, сходимость через F, коллективный иммунитет.

3. **Доказательства переведены на строгую математическую основу:**
   - Теорема Колмогорова–Тихомирова об ε-энтропии.
   - Принцип максимума Понтрягина для оптимального управления.
   - Теория ветвящихся процессов Севастьянова для иммунитета роя.

4. **D_f сохранён как эвристический индикатор в TUI, но исключён из всех математических доказательств.**

## Приложение A. Формулы и алгоритмы

### A.1. Вычисление Intrinsic Dimension (ID)

Алгоритм локального PCA с адаптивным радиусом (Фукунага–Олсен–Погорелов):

1. Для каждой точки \(z_i\) в латентном пространстве построить k ближайших соседей (\(k \approx 50\)).
2. Вычислить локальную ковариационную матрицу \(C_i\) размера \(d \times d\).
3. Найти сингулярные числа \(\sigma_1 \ge \sigma_2 \ge ... \ge \sigma_d\).
4. Определить порог \(\tau\) из распределения Марченко–Пастура для шумовых компонент.
5. Локальная размерность \(ID_i = \#\{j : \sigma_j > \tau\}\).
6. Глобальная ID = медиана локальных оценок.

Сложность: \(O(N \log N)\) с HNSW-индексом. На GPU для \(N = 10^5\) и \(d = 1280\) время ~30 секунд.

### A.2. Вычисление свободной энергии

\[
F(P_t) = \text{CrossEntropy}(P_t) + \lambda \cdot \text{Complexity}(P_t)
\]

где:
- CrossEntropy = \(-\log P_t(y_{true} | x)\) с температурным масштабированием.
- Complexity = \(\sum_i \mathbf{1}_{w_i > \theta}\) — число активных записей в SDM.
- \(\lambda\) подбирается так, чтобы оба слагаемых были одного порядка на старте.

### A.3. Эмиссия FFF по ΔF

\[
E_{mission} = k \cdot \max(0, F_{before} - F_{after})
\]

где k — глобальный коэффициент конвертации.

### A.4. Условие коллективного иммунитета

\[
R_0 = \lambda_1 \cdot p \cdot (1 - \beta) < 1
\]

где:
- \(\lambda_1\) — наибольшее собственное значение графа связей нод.
- \(p\) — вероятность генерации ложного чанка.
- \(\beta\) — точность локального теста (Bongard+).

При \(R_0 < 1\) вероятность эпидемии ложных знаний экспоненциально мала.

### A.5. Каскад Фейгенбаума (эвристика)

Сохранён как индикатор в TUI, но исключён из доказательств. Вычисляется как:

\[
\delta_n = \frac{D_f^{(n)} - D_f^{(n-1)}}{D_f^{(n+1)} - D_f^{(n)}}
\]

Если \(\delta_n \to 4.669...\) — предвестник хаотизации (сигнал тревоги).

### 11.2. Принципы архитектуры v2.0
- **Kolmogorov-минимум:** Минимальное описание через сжатие ID и дистилляцию LeWM.
- **Pontryagin-оптимум:** Ресурсы направляются на минимизацию F, стратегия всегда экстремальна.
- **Lyapunov-устойчивость:** Свободная энергия F — функция Ляпунова. Система стабильна при dF/dt < 0.
- **Turchin-эмерджентность:** Каждый скачок ID порождает новые свойства. Рой — не сумма нод, а новый разум.

## Часть 12. Философия и алгоритмы малых вычислений
### 12.1. Философский фундамент
Система WATERS стоит на плечах гигантов:
- **Turing (1936):** Разделение на быстрый неполный ASP и медленный полный LLM. Понимание пределов вычислимости.
- **Shannon (1948):** Энтропия как мера неопределённости знаний. Диагностика "здоровья" агента.
- **Wiener (1948):** Кибернетический цикл: [Действие] → [Измерение] → [Сравнение] → [Коррекция].
- **Von Neumann (1966):** Самовоспроизведение нод через геном. Бессмертие знаний.
- **Mandelbrot (1975):** Фрактальная структура иерархии знаний. Экономия памяти через самоподобие.
- **Euler (1736):** Теория графов для анализа связности Knowledge Fabric.

### 12.2. Карта малых вычислений
| Алгоритм | Сложность | RAM | Что даёт |
|----------|-----------|-----|----------|
| SDM Kanerva | O(log N) | 500 MB | 80% hit rate без LLM |
| Ситуатор (Pospelov) | O(K) | 50 MB | Контекст из запроса |
| LeWM инференс (CPU) | O(d²) | 10-50 MB | Внутренняя интуиция |
| Оценка ID | O(N log N) | 100 MB | Зрелость памяти |
| Расчёт F | O(1) | 1 KB | Здоровье агента |
| Bongard+ | O(R) | 20 MB | Верификация |
| Federator | O(1) | 1 KB | Дофамин |
| Прионный обмен | O(H) | 100 KB | Распространение знаний |
| **Итого на ноде** | | **~580 MB** | **Полноценный когнитивный агент** |

### 12.3. Обратный Scaling Law
Современный ML: Больше данных + compute + параметры = лучше качество.
WATERS: Меньше данных + compute + умнее алгоритмы = достаточно качества для 80% задач. Это наследие советской школы малых вычислений.

## Приложение F. Словарь терминов
- **ID (Intrinsic Dimension)**: Строгая мера структурной сложности многообразия данных.
- **F (Free Energy)**: Глобальный функционал (ошибка + сложность), минимизируемый агентом.
- **L_pred**: Ошибка предсказания модели мира LeWM (MSE в латентном пространстве).
- **Divergence**: Мера расхождения между моделями мира разных нод.
- **LeWM**: Модель мира JEPA, реализующая внутреннее предвидение.
- **GDSL**: Язык описания алгоритмов (Glushkov Domain-Specific Language).
- **FFF**: Валюта значимости, эмитируемая под снижение F.
