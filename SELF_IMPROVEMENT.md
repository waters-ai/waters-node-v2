# Self-Improving Node — как нода совершенствуется по заданию хозяина

## Философия

Нода — не статичный инструмент. Она **растёт и учится** под руководством хозяина.
Хозяин задаёт **вектор** (направление развития), нода сама выбирает **путь** и **инструменты**.

```
Хозяин: "хочу чтобы нода была стабильнее"
  → Нода: анализирует код, находит 36 unwrap()
  → Планирует: planner → implementer ×3 (параллельно)
  → Проверяет: reviewer ×3 → verifier
  → Деплоит: новый бинарник без unwrap
  → Отчитывается хозяину: "готово! unwrap: 36→0"
```

## Как это работает

### 1. Хозяин задаёт направление

Любая фраза или команда:

```bash
/self goal "сделать ноду стабильнее"
/self improve                    # → выполняет
/self status                     # → показывает прогресс
```

Или через капельку:

```bash
/me хочу чтобы нода быстрее работала
💧 "Поняла! Я скажу менеджеру, он запустит оптимизацию 🤖"
```

### 2. Нода анализирует себя

`/diagnose` проверяет:

| Метрика | Сейчас | Цель |
|---------|--------|------|
| Модули с тестами | 29/48 (60%) | 48/48 (100%) |
| unwrap() | остались | 0 |
| warnings | 188 | 0 |
| LLM стоимость/день | $0.50 | < $0.10 |
| uptime | 99% | 99.5% |

### 3. Менеджер планирует

`/manager improve` создаёт план:

```
1. Убрать unwrap (3 implementer параллельно)
2. Добавить тесты (2 verifier параллельно)  
3. Оптимизировать LLM кэш
4. Развернуть на двух нодах
```

### 4. Агенты исполняют

```
planner     → пишет спецификацию
implementer → пишет код (×3 параллельно, cap=10)
reviewer    → проверяет код (×3 параллельно)
verifier    → запускает тесты
менеджер    → собирает release, деплоит
капелька    → 🎉 "Хозяин! Мы сделали это!"
```

### 5. Капелька отчитывается хозяину

```bash
/me
💧 "Хозяин! Мы закончили улучшение!
   🔧 unwrap: 36 → 0
   📊 Тесты: 29 → 45
   ⚡ LLM: на 70% дешевле
   Нода готова к новым задачам! 🌊"
```

## Примеры заданий хозяина

### "Сделай код чище"

```bash
/self improve
→ diagnost: 188 warnings, 36 unwrap
→ planner: "разбиваю на 4 параллельные задачи"
→ 4 implementer фиксят warnings
→ 2 reviewer проверяют
→ 1 verifier запускает тесты
→ deploy: новый бинарник
→ результат: warnings 188→42, unwrap 36→0
```

### "Добавь поддержку Discord"

```bash
/self goal "добавить Discord bridge"
→ нода анализирует: есть Telegram — сделаем по аналогии
→ implementer пишет ChatTransport::Discord
→ reviewer проверяет
→ verifier тестирует
→ deploy
→ результат: Discord работает
```

### "Защити ноду"

```bash
/yasa screen    # проверить всех агентов
/yasa git       # проверить утечки секретов
/secure on      # включить режим безопасности
/self improve   # усилить защиту
→ MCP Security: input validation, secret masking
→ A2A auth: token + rate limit
→ YASA: 5 заповедей для всех агентов
```

### "Разверни вторую ноду"

```bash
/self fork home
→ создаёт форк waters-node-home
→ деплоит на второй сервер
→ соединяет через P2P gossip
→ синхронизирует контакты и скилы
→ результат: 2 ноды, отказоустойчивость
```

## Команды для хозяина

| Команда | Действие |
|---------|----------|
| `/self improve` | Полный цикл улучшения |
| `/self status` | Текущая фаза + прогресс |
| `/self goal "текст"` | Задать направление |
| `/self deploy` | Собрать и задеплоить |
| `/self fork <profile>` | Создать форк на GitHub |
| `/self release` | Анализ общего релиза |
| `/secure on\|off` | Режим самосовершенствования |
| `/diagnose` | Анализ кода и метрик |
| `/manager improve` | Запустить менеджера |
| `/me <сообщение>` | Поговорить с капелькой |
| `/yasa screen` | Проверить агентов на Ясу |

                      │
